# 주식 AI 분석 시스템 - Ubuntu 구축 가이드

## 0. 전체 구조

```
stock_analyzer/
├── main.py                        # 진입점 (CLI)
├── requirements.txt               # 의존성
├── .env.example                   # API 키 템플릿
├── .env                           # 실제 API 키 (직접 생성)
├── config/
│   └── settings.py                # 전역 설정 (리스크 파라미터, 지표 파라미터)
├── core/
│   ├── data_collector.py          # 데이터 수집 (yfinance, FRED, 옵션, 내부자거래)
│   └── indicators.py              # 기술 지표 계산 (SMA, EMA, RSI, MACD, BB, ADX, ATR, OBV)
├── risk/
│   └── risk_manager.py            # 리스크 관리 (포지션사이징, 손절/익절, Kelly)
├── visualization/
│   └── chart_generator.py         # 차트 생성 (Plotly 기본 + matplotlib fallback)
├── analysis/
│   └── ai_analyzer.py             # AI 분석 (GPT-4o 멀티모달, Ollama 로컬)
└── output/                        # 리포트 출력 폴더 (자동 생성)
```

## 1. Ubuntu 환경 준비

### 1.1 Python 환경

```bash
# Python 3.10+ 확인
python3 --version

# 가상환경 생성 (권장)
cd ~/stock_analyzer
python3 -m venv venv
source venv/bin/activate

# 의존성 설치
pip install -r requirements.txt
```

### 1.2 Plotly 차트용 Chrome 설치 (선택)

Plotly의 `kaleido`가 고해상도 PNG 차트를 생성하려면 Chrome이 필요하다. 없으면 자동으로 matplotlib fallback이 작동하므로 필수는 아니다.

```bash
# Plotly 내장 명령으로 설치 (가장 간단)
plotly_get_chrome

# 또는 수동 설치
wget https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb
sudo dpkg -i google-chrome-stable_current_amd64.deb
sudo apt-get install -f
```

Plotly 차트가 실패하면 matplotlib가 자동 대체한다. 품질 차이:

| 항목 | Plotly + Chrome | matplotlib (fallback) |
|------|----------------|----------------------|
| 캔들차트 | O (네이티브) | X (종가 라인으로 대체) |
| 해상도 | 2800x2000 | 2100x1500 |
| AI 분석 품질 | 패턴 인식 우수 | 충분히 사용 가능 |
| 설치 난이도 | Chrome 필요 | 추가 설치 없음 |

## 2. API 키 설정

### 2.1 .env 파일 생성

```bash
cd ~/stock_analyzer
cp .env.example .env
nano .env   # 또는 vim .env
```

### 2.2 필수/선택 API 키

```
# ── 무료 (필수 아님, 있으면 거시경제 데이터 추가) ──
FRED_API_KEY=xxxxx

# ── 유료 (AI 분석 사용 시) ──
OPENAI_API_KEY=sk-xxxxx

# ── 로컬 LLM (Ollama 사용 시) ──
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=llama3.1:8b
```

**API 키 없이도 기본 분석은 동작한다.** yfinance는 API 키 불필요.

### 2.3 각 API 키 발급 방법

**FRED (무료, 1분 소요)**
1. https://fred.stlouisfed.org/docs/api/api_key.html 접속
2. "Request API Key" 클릭
3. 이메일 인증 후 즉시 발급

**OpenAI (유료, GPT-4o 사용 시)**
1. https://platform.openai.com/api-keys 접속
2. 계정 생성 → API Key 생성
3. 결제 수단 등록 필요
4. 비용: 일 10회 분석 기준 약 $5-15/월

## 3. 실행 방법

### 3.1 기본 실행 (AI 없이)

```bash
cd ~/stock_analyzer
source venv/bin/activate

# 단일 종목 분석
python main.py AAPL

# 계좌 크기 지정 (리스크 계산에 반영)
python main.py NVDA --account 50000
```

출력물:
- `output/AAPL_report_20260403.json` — 구조화된 전체 데이터
- `output/AAPL_report_20260403.txt` — 사람이 읽기 좋은 텍스트 리포트
- `output/AAPL_analysis.png` — 기술 분석 차트 이미지

### 3.2 GPT-4o 멀티모달 분석

```bash
# 차트 이미지 + 수치 데이터를 GPT-4o에 동시 전송
python main.py AAPL --ai gpt4o --account 30000
```

GPT-4o는 차트 이미지에서 패턴(헤드앤숄더, 더블바텀 등)을 시각적으로 인식하고, 수치 데이터와 교차 검증하여 종합 판단을 내린다.

### 3.3 Ollama 로컬 LLM 분석

```bash
# Ollama 먼저 설치/실행
curl -fsSL https://ollama.ai/install.sh | sh
ollama pull llama3.1:8b    # 8B 모델 (VRAM 8GB)
ollama pull llama3.1:70b   # 70B 모델 (VRAM 48GB, 품질 우수)
ollama serve               # 백그라운드 실행

# 분석 실행
python main.py MSFT --ai ollama
```

Ollama는 텍스트 기반 분석만 수행한다 (차트 이미지 미지원). 멀티모달이 필요하면 GPT-4o를 사용해야 한다.

### 3.4 여러 종목 일괄 분석 (셸 스크립트)

```bash
#!/bin/bash
# batch_analyze.sh
TICKERS="AAPL MSFT NVDA GOOGL AMZN TSLA META"
for t in $TICKERS; do
    echo "=== $t 분석 시작 ==="
    python main.py $t --ai gpt4o --account 50000
    sleep 2  # API rate limit 방지
done
echo "=== 전체 완료 ==="
```

```bash
chmod +x batch_analyze.sh
./batch_analyze.sh
```

## 4. cron 자동 스케줄링 (매일 자동 분석)

### 4.1 cron 등록

```bash
crontab -e
```

아래 내용 추가:

```cron
# 매일 미국 장 마감 후 실행 (한국 시간 06:00 = 미 동부 17:00)
0 6 * * 2-6 /home/사용자명/stock_analyzer/venv/bin/python /home/사용자명/stock_analyzer/main.py AAPL --ai gpt4o --account 50000 >> /home/사용자명/stock_analyzer/output/cron.log 2>&1

# 여러 종목
5 6 * * 2-6 /home/사용자명/stock_analyzer/batch_analyze.sh >> /home/사용자명/stock_analyzer/output/cron.log 2>&1
```

### 4.2 systemd timer (cron 대안, 더 안정적)

```bash
# /etc/systemd/system/stock-analyzer.service
sudo nano /etc/systemd/system/stock-analyzer.service
```

```ini
[Unit]
Description=Stock Analyzer Daily Run
After=network-online.target
Wants=network-online.target

[Service]
Type=oneshot
User=사용자명
WorkingDirectory=/home/사용자명/stock_analyzer
ExecStart=/home/사용자명/stock_analyzer/venv/bin/python main.py AAPL --ai gpt4o --account 50000
StandardOutput=append:/home/사용자명/stock_analyzer/output/systemd.log
StandardError=append:/home/사용자명/stock_analyzer/output/systemd.log

[Install]
WantedBy=multi-user.target
```

```bash
# /etc/systemd/system/stock-analyzer.timer
sudo nano /etc/systemd/system/stock-analyzer.timer
```

```ini
[Unit]
Description=Run Stock Analyzer daily

[Timer]
OnCalendar=Tue..Sat 06:00
Persistent=true

[Install]
WantedBy=timers.target
```

```bash
sudo systemctl enable stock-analyzer.timer
sudo systemctl start stock-analyzer.timer
sudo systemctl list-timers  # 확인
```

## 5. Ollama 로컬 LLM 상세 설정

### 5.1 설치


### 5.2 모델 선택 가이드

| 모델 | VRAM 필요 | 분석 품질 | 속도 |
|------|-----------|-----------|------|
| `llama3.1:8b` | 8GB | 보통 | 빠름 |
| `llama3.1:70b` | 48GB | 좋음 | 느림 |
| `qwen2.5:32b` | 24GB | 좋음 | 보통 |
| `deepseek-v2:16b` | 12GB | 수치 분석 우수 | 보통 |
| `llava:34b` | 24GB | 멀티모달 가능 | 느림 |

GPU 없이 CPU만으로도 동작하지만 8B 모델 기준 응답에 30초-2분 소요.

### 5.3 백그라운드 서비스 등록

### 5.4 .env 설정

## 6. 설정 커스터마이징

### 6.1 리스크 파라미터 조정

`config/settings.py` 파일에서 수정:

```python
# 공격적 매매자
MAX_POSITION_PCT = 0.10          # 단일 종목 최대 10%
DEFAULT_STOP_LOSS_ATR_MULT = 1.5 # ATR 1.5배 (좁은 손절)
DEFAULT_TAKE_PROFIT_RATIO = 3.0  # R:R = 1:3

# 보수적 매매자
MAX_POSITION_PCT = 0.03          # 단일 종목 최대 3%
DEFAULT_STOP_LOSS_ATR_MULT = 2.5 # ATR 2.5배 (넓은 손절)
DEFAULT_TAKE_PROFIT_RATIO = 1.5  # R:R = 1:1.5

# 계좌 크기
ACCOUNT_SIZE = 50000             # 기본값 변경 (또는 --account 옵션 사용)
```

### 6.2 기술 지표 파라미터 조정

```python
# 단기 매매자
```

## 7. 출력물 해석 방법

### 7.1 콘솔 출력 예시

```
============================================================
  AAPL 종합 분석 시작 - 2026-04-03 18:30
============================================================

[1/6] 데이터 수집 중...
  ✓ OHLCV: 503일 데이터 수집 완료
  ✓ 펀더멘털: Apple Inc.
  ✓ 옵션: P/C Ratio(Vol)=0.82
  ✓ 거시경제: 7개 지표 수집

[2/6] 기술 지표 계산 중...
  ✓ 현재가: $178.52
  ✓ RSI: 62.3
  ✓ 추세 강도: strong

[3/6] 리스크 분석 중...
  ✓ 손절가: $170.14 (4.69%)
  ✓ 익절가: $195.28 (9.39%)
  ✓ 권장 수량: 23주 ($4105.96)
```

### 7.2 핵심 지표 해석

| 지표 | 값 | 의미 |
|------|-----|------|
| RSI > 70 | 과매수 | 단기 하락 가능성. 신규 매수 주의 |
| RSI < 30 | 과매도 | 반등 가능성. 매수 검토 |
| ADX > 25 | 강한 추세 | 추세 추종 전략 유효 |
| ADX < 20 | 추세 없음 | 횡보장. 레인지 매매 또는 관망 |
| P/C Ratio > 1.0 | 풋 우위 | 시장 공포 심리. 역발상 매수 신호 가능 |
| P/C Ratio < 0.7 | 콜 우위 | 시장 낙관. 과열 주의 |
| BB position: above_upper | 밴드 상단 돌파 | 과매수 또는 강한 모멘텀 |
| BB position: below_lower | 밴드 하단 돌파 | 과매도 또는 강한 하락 모멘텀 |

### 7.3 리스크 관리 수치 해석

- **손절가**: 이 가격에 도달하면 무조건 청산. ATR 2배 거리가 기본
- **익절가**: 목표 수익 달성 시 일부/전체 청산
- **R:R 비율**: 1:2 이상이면 승률 40%만 되어도 장기 수익 가능
- **포지션 비중**: 5% 이하 유지가 원칙. 초과 시 경고 출력

## 9. 문제 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| `OHLCV 데이터 수집 실패` | yfinance 네트워크 오류 또는 잘못된 티커 | 인터넷 연결 확인. 티커명 확인 (예: BRK.B, META) |
| `차트 생성 실패` | Chrome/kaleido 미설치 | `plotly_get_chrome` 실행하거나, 무시해도 matplotlib로 자동 대체 |
| `FRED 수집 실패` | API 키 없음 또는 만료 | .env 파일에 FRED_API_KEY 확인 |
| `GPT-4o 분석 오류` | API 키 또는 잔액 부족 | OpenAI 대시보드에서 키/잔액 확인 |
| `Ollama 연결 불가` | Ollama 서버 미실행 | `ollama serve` 실행 또는 `systemctl start ollama` |
| `ImportError: No module` | 의존성 미설치 | `pip install -r requirements.txt` 재실행 |
| `Permission denied` | 가상환경 미활성화 | `source venv/bin/activate` 실행 |
