## 시스템 개요

Stock AI WebUI는 16개 분석 도구와 6개 전문 AI 에이전트가 협업하여 종합적인 주식 분석을 제공하는 웹 인터페이스입니다.

### 주요 특징
- **실시간 분석**: Watchlist 종목 자동 스캔 (30분 주기)
- **AI 협업**: Multi-Agent 시스템으로 다각도 분석
- **백테스트**: 4가지 전략의 과거 수익률 검증
- **ML 예측**: 5개 모델 앙상블로 미래 방향성 예측
- **모의매매**: Paper Trading으로 전략 검증

## WebUI 접속 방법

### 1. 서비스 시작

```bash
# 터미널 1: FastAPI 서버 시작
cd ~/stock_auto/chart_agent_service
python service.py

# 터미널 2: WebUI 시작
cd ~/stock_auto/stock_analyzer
streamlit run webui.py --server.port 8501
```

### 2. 브라우저 접속

```
http://localhost:8501
```

원격 접속 시:
```
http://[서버IP]:8501
```

## 2. Dashboard - 종합 대시보드

### 📌 주요 기능
- 전체 Watchlist 종목 현황 한눈에 파악
- 시장 지수 실시간 확인
- 최신 분석 신호 확인
- 신뢰도별 필터링

### 🖥️ 화면 구성

#### 상단: 시장 지수
```
┌──────┬──────┬──────┬──────┬──────┐
│ S&P  │ QQQ  │ VIX  │ DXY  │ 10Y  │
│+0.8% │+1.2% │15.2  │102.3 │4.21% │
└──────┴──────┴──────┴──────┴──────┘
```

#### 중앙: Watchlist 종목 카드
```
┌─────────────────┬─────────────────┐
│     AAPL        │      NVDA       │
│   Signal: BUY   │  Signal: HOLD   │
│   Score: +6.2   │   Score: +1.5   │
│  Conf: 8/10     │   Conf: 5/10    │
│  ⬆ +2.3%        │   ⬆ +0.8%       │
└─────────────────┴─────────────────┘
```

### 📊 지표 해석

| 지표 | 의미 | 해석 방법 |
|------|------|-----------|
| **Signal** | 최종 매매 신호 | BUY(매수), SELL(매도), HOLD(관망) |
| **Score** | 종합 점수 | -10 ~ +10, ±5 이상 강한 신호 |
| **Confidence** | 신뢰도 | 0-10, 7 이상 신뢰할 만함 |
| **% Change** | 당일 변동률 | 빨강(하락), 초록(상승) |

### 💡 활용 팁
- 신뢰도 7 이상 + Score ±5 이상 종목에 주목
- 카드 클릭 시 해당 종목 Detail 페이지로 이동
- 우측 필터로 BUY/SELL 신호만 보기 가능

## 3. Detail - 종목 상세 분석

### 📌 주요 기능
- 16개 분석 도구 상세 결과
- 기술적 차트 이미지
- 도구별 신호와 점수
- AI 종합 판단

### 🖥️ 화면 구성

#### 상단: 종목 선택 및 기본 정보
```
종목 선택: [AAPL ▼] [🔄 새로고침]

현재가: $178.52 (+2.3%)
최종 신호: BUY (신뢰도: 8/10)
```

#### 중앙: 분석 차트
```
┌─────────────────────────────────────┐
│  [차트 이미지]                       │
│  • 가격 + 이동평균선                 │
│  • 거래량                           │
│  • RSI                              │
│  • MACD                             │
└─────────────────────────────────────┘
```

#### 하단: 16개 도구 상세 결과
```
┌─────────────────────────────────────┐
│ 📈 기술적 분석 (6개)                 │
├─────────────────────────────────────┤
│ • trend_ma: BUY (+2.0)              │
│   → SMA 정배열, 골든크로스           │
│ • rsi_divergence: NEUTRAL (0)       │
│   → 다이버전스 없음                  │
│ • bollinger_squeeze: BUY (+1.5)     │
│   → 스퀴즈 후 상방 돌파              │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ 📊 퀀트 분석 (6개)                   │
├─────────────────────────────────────┤
│ • fibonacci: BUY (+1.0)             │
│   → 38.2% 되돌림 후 반등            │
│ • momentum_rank: BUY (+2.0)         │
│   → 3개월 > 1개월 모멘텀            │
└─────────────────────────────────────┘
```

### 📊 도구별 신호 해석

각 도구는 3가지 신호를 제공:
- **BUY** (녹색): 매수 신호
- **SELL** (빨강): 매도 신호
- **NEUTRAL** (회색): 중립/불확실

점수 범위: -3.0 ~ +3.0
- ±2.0 이상: 강한 신호
- ±1.0~2.0: 보통 신호
- ±1.0 미만: 약한 신호

### 💡 활용 팁
- 여러 도구가 같은 방향 신호 → 신뢰도 높음
- 기술적 + 퀀트가 일치할 때 특히 주목
- 차트 이미지 다운로드 가능 (우클릭 → 저장)

### 👥 6개 에이전트 역할

| 에이전트 | 담당 분야 | 사용 LLM |
|----------|----------|----------|
| **Technical Analyst** | 차트 패턴, 기술 지표 | Gemini |
| **Quant Analyst** | 통계 모델, 수학적 분석 | Gemini |
| **Risk Manager** | 리스크 평가, 포지션 사이징 | Ollama |
| **ML Specialist** | 머신러닝 예측 | Ollama |
| **Event Analyst** | 뉴스, 이벤트 영향 | Gemini |
| **Decision Maker** | 최종 판단, 충돌 해결 | GPT-4o |

### 📥 Export 기능

결과를 3가지 형식으로 다운로드:
- **JSON**: 프로그램 연동용
- **CSV**: 엑셀 분석용
- **Markdown**: 문서/리포트용

### 💡 활용 팁
- 의견이 4:1 이상으로 일치 → 강한 신호
- Decision Maker의 최종 의견이 가장 중요
- 소수 의견도 리스크 파악에 유용

## 5. Scan Log - 스캔 로그

### 📌 주요 기능
- 자동 스캔 이력 조회
- 주간 트렌드 분석
- 종목별 신호 변화 추적
- 도구별 성과 분석

### 🖥️ 화면 구성

#### 탭 구조
```
[최근 스캔] [종목별 이력] [주간 요약] [도구별 트렌드]
```

#### 최근 스캔 로그
```
┌────────────────┬──────┬───────┬─────────┐
│ 시간            │ 종목  │ 신호   │ 신뢰도   │
├────────────────┼──────┼───────┼─────────┤
│ 10:30:00       │ AAPL │ BUY   │ 8/10    │
│ 10:30:15       │ NVDA │ HOLD  │ 5/10    │
│ 10:30:30       │ TSLA │ SELL  │ 7/10    │
└────────────────┴──────┴───────┴─────────┘
```

#### 주간 트렌드 차트
```
AAPL 주간 추이
       월  화  수  목  금
Score: 3.5 4.2 5.1 6.2 6.8 ↗
Signal: H   H   B   B   B
```

### 📊 트렌드 분석 해석
- **상승 추세 (↗)**: 점수가 지속 상승
- **전환 시점**: HOLD → BUY 전환 포착
- **일관성**: 신호가 자주 바뀌면 불안정

### 💡 활용 팁
- 3일 이상 같은 신호 유지 → 신뢰도 높음
- 주간 점수 상승 추세 → 모멘텀 강화
- 도구별 적중률로 신뢰할 도구 파악

## 6. Backtest - 백테스트

### 📌 주요 기능
- 4가지 전략 과거 검증
- HyperOpt 파라미터 최적화
- Walk-Forward 과적합 방지
- 수익 곡선 시각화

### 🖥️ 화면 구성

#### 전략 선택
```
종목: [AAPL ▼]
전략: ○ SMA Cross  ● RSI Reversion  ○ Bollinger  ○ Composite
기간: [2년 ▼]  [백테스트 실행]
```

#### 결과 요약
```
┌─────────────────────────────────────┐
│ 📈 백테스트 결과                     │
├─────────────────────────────────────┤
│ • 연간 수익률: 28.5%                 │
│ • Sharpe Ratio: 2.34               │
│ • Max Drawdown: -12.3%             │
│ • 승률: 58.3%                      │
│ • 평균 보유기간: 5.2일               │
└─────────────────────────────────────┘
```

#### 수익 곡선 그래프
```
Portfolio Value
   ^
120K│      ╱╲    ╱
110K│     ╱  ╲__╱
100K│────╯
 90K│
    └──────────────────> Time
      Jan  Mar  May  Jul
```

### 🔧 HyperOpt 최적화

Optuna로 자동 파라미터 탐색:
```
최적 파라미터 발견!
• RSI 임계값: 32 (기존 30)
• 보유 기간: 5일 (기존 7일)
→ Sharpe Ratio: 1.8 → 2.34 (+30%)
```

### 📊 주요 지표 해석

| 지표 | 좋음 | 보통 | 나쁨 |
|------|-----|-----|-----|
| **Sharpe Ratio** | > 2.0 | 1.0-2.0 | < 1.0 |
| **Max Drawdown** | < -10% | -10~20% | > -20% |
| **승률** | > 60% | 50-60% | < 50% |
| **연간 수익률** | > 20% | 10-20% | < 10% |

### 💡 활용 팁
- Walk-Forward로 과적합 여부 확인 필수
- Sharpe Ratio 우선 (수익률보다 중요)
- Drawdown -20% 넘으면 리스크 과다

## 7. ML Predict - 머신러닝 예측

#### 모델별 예측
```
┌──────────────┬────────┬─────────┐
```

### 🧠 모델 설명

| 모델 | 특징 | 장점 |
|------|------|------|

| **XGBoost** | 부스팅 | 높은 정확도 |
| **LightGBM** | 경량 부스팅 | 빠른 속도 |

| **GradientBoost** | 순차 부스팅 | 정밀 조정 |

### 💡 활용 팁
- 4개 이상 모델이 같은 방향 → 신뢰도 높음
- SHAP Top 3 feature 확인 → 핵심 변수 파악
- LSTM이 다른 예측 → 단기 패턴 변화 가능성

## 8. Portfolio - 포트폴리오 최적화

### 📌 주요 기능
- Markowitz 평균-분산 최적화
- Risk Parity 균등 리스크 배분
- 효율적 프론티어 시각화
- 상관관계 히트맵

### 🖥️ 화면 구성

#### 최적화 방법 선택
```
방법: ● Markowitz  ○ Risk Parity
목표 수익률: [15% ▼]
[포트폴리오 최적화]
```

#### 최적 배분 결과
```
┌─────────────────────────────────────┐
│ 📊 최적 포트폴리오                   │
├──────┬──────────┬─────────┬────────┤
│ 종목  │ 비중      │ 금액     │ 주수   │
├──────┼──────────┼─────────┼────────┤
│ AAPL │ 35%      │ $35,000 │ 196주  │
│ MSFT │ 25%      │ $25,000 │ 62주   │
│ NVDA │ 20%      │ $20,000 │ 23주   │
│ GOOGL│ 20%      │ $20,000 │ 140주  │
└──────┴──────────┴─────────┴────────┘
```

#### 포트폴리오 성과 지표

```
예상 연간 수익률: 18.5%

Sharpe Ratio: 1.14
최대 손실(95% VaR): -8.3%
```

### 📈 효율적 프론티어

### 🔧 최적화 방법 비교

| 방법 | 특징 | 적합한 경우 |
|------|------|------------|
| **Markowitz** | 수익 극대화, 리스크 최소화 | 목표 수익률이 명확할 때 |
| **Risk Parity** | 모든 종목 동일 리스크 기여 | 안정적 분산 투자 원할 때 |

### 💡 활용 팁
- 상관관계 낮은 종목 조합 → 리스크 감소
- Sharpe Ratio 1.0 이상 목표
- 한 종목 40% 초과 비중 피하기

## 10. Paper Trade - 모의매매

### 📌 주요 기능
- 실제 자금 없이 매매 시뮬레이션
- 자동/수동 주문 실행
- 포지션 추적 및 손익 계산
- Trailing Stop 자동 청산

### 🖥️ 화면 구성

#### 포트폴리오 현황
```
┌─────────┬─────────┬─────────┬─────────┐
│총 자산   │ 현금     │ 손익     │ 수익률   │
│$105,230 │$52,100  │+$5,230  │+5.23%   │
└─────────┴─────────┴─────────┴─────────┘
```

#### 보유 포지션
```
┌──────┬────┬──────┬──────┬───────┬──────┐
│종목   │수량│진입가 │현재가 │손익    │수익률 │
├──────┼────┼──────┼──────┼───────┼──────┤
│AAPL  │100 │$175  │$180  │+$500  │+2.9% │
│NVDA  │ 50 │$850  │$865  │+$750  │+1.8% │
└──────┴────┴──────┴──────┴───────┴──────┘
```

#### 주문 입력
```
[자동 매매] [수동 주문]

수동 주문:
종목: [TSLA___]  액션: [BUY ▼]
수량: [10]       가격: [$220.50]
[주문 실행]
```

### 🎯 자동 매매 로직

1. 최신 분석 결과 확인
2. 조건 충족 시 자동 실행:
   - BUY 신호 + 신뢰도 ≥ 5 → 매수
   - SELL 신호 → 보유 종목 매도
3. 리스크 관리:
   - Trailing Stop: 고점 대비 -5% 자동 청산
   - Stop Loss: 진입가 대비 -3% 손절
   - Time Stop: 10일 경과 시 청산

### 📊 성과 지표

| 지표 | 의미 | 좋음 |
|------|------|------|
| **Win Rate** | 수익 거래 비율 | > 55% |
| **Profit Factor** | 총이익/총손실 | > 1.5 |
| **Max Drawdown** | 최대 손실폭 | < -10% |

### 💡 활용 팁
- 실제 투자 전 전략 검증 필수
- 최소 100회 거래 후 평가
- 초기 자본 $100,000으로 시작

## 자주 묻는 질문

### Q1: 신뢰도가 낮게 나오는 이유?
**A:** 시장이 혼조세이거나 데이터가 상충할 때 발생. 신뢰도 5 이하는 관망 권장.

### Q2: Multi-Agent가 느린 이유?
**A:** 6개 에이전트가 병렬로 작업하지만 각자 LLM 호출하므로 90초 정도 소요. 정확도를 위한 트레이드오프.

### Q3: Paper Trade 리셋 방법?
**A:** Paper Trade 페이지 하단 [Reset Paper Trading] 버튼 클릭.

### Q4: 백테스트 과적합 확인?
**A:** Walk-Forward 탭에서 각 구간 Sharpe Ratio 확인. 편차가 크면 과적합.

### Q5: API 에러 발생 시?
**A:**
1. FastAPI 서버 실행 확인 (http://localhost:8100)
2. 사이드바 [Restart Agent] 클릭
3. 터미널에서 서비스 재시작

### Q6: 차트가 안 보일 때?
**A:**
1. 종목 분석이 완료되었는지 확인
2. output/ 폴더에 PNG 파일 존재 확인
3. 브라우저 캐시 새로고침 (Ctrl+F5)

### Q7: 실시간 데이터인가요?
**A:** 아닙니다. yfinance는 15-20분 지연 데이터. 장 마감 후 분석에 최적화.

### Q8: 모바일에서 사용 가능?
**A:** 가능하나 PC 환경 권장. 차트와 테이블이 모바일에서는 작게 표시됨.
