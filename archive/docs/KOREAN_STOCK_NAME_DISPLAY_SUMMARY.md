# 한국 주식 종목명 표시 개선 요약

## 개선 내용

멀티에이전트 시스템에서 한국 주식이 숫자 코드(예: 005930)로만 표시되던 것을 종목명과 함께 표시하도록 개선했습니다.

## 주요 변경 사항

### 1. **multi_agent.py 수정**

#### MultiAgentOrchestrator 클래스
- `_get_stock_name()` 메서드 추가: 티커로부터 종목명을 가져오는 기능
- `analyze()` 메서드: 분석 시작 시 종목명 표시
  - 변경 전: `[MultiAgent] 005930.KS 분석 시작...`
  - 변경 후: `[MultiAgent] 삼성전자(005930.KS) 분석 시작...`

#### BaseAgent 클래스
- `_get_stock_name_for_agent()` 메서드 추가: 에이전트용 간단한 종목명 조회
- `_build_prompt()` 메서드: LLM 프롬프트에 종목명 포함
  - 변경 전: `005930.KS 종목에 대한 분석...`
  - 변경 후: `삼성전자(005930.KS) 종목에 대한 분석...`

#### 특수 에이전트들
- MLSpecialist, GeopoliticalAnalyst, ValueInvestor의 `analyze()` 메서드에서도 종목명 표시

#### DecisionMaker 클래스
- `_build_decision_prompt()` 메서드: 최종 의사결정 프롬프트에 종목명 포함

### 2. **korean_stock_favorites.json 확장**

기존 7개 종목에서 58개 주요 종목으로 확대:

```json
{
  "005930": "삼성전자",
  "000660": "SK하이닉스",
  "035720": "카카오",
  "035420": "네이버",
  "373220": "LG에너지솔루션",
  "068270": "셀트리온",
  "005490": "포스코홀딩스",
  // ... 51개 추가 종목
}
```

### 3. **종목명 조회 우선순위**

1. korean_stocks.py의 KoreanStockData.get_stock_name() 메서드
2. korean_stock_favorites.json 파일 직접 조회
3. yfinance API를 통한 종목명 조회

## 사용 예시

### 분석 시작 메시지
```
[MultiAgent] 삼성전자(005930.KS) 분석 시작...
```

### 에이전트 프롬프트
```
당신은 Technical Analyst 전문가입니다.
SK하이닉스(000660.KS) 종목에 대한 분석 결과를 해석하고...
```

### Decision Maker 프롬프트
```
당신은 최종 의사결정자입니다.
여러 전문가의 LG에너지솔루션(373220.KS) 분석 결과를 종합하여...
```

## 테스트 결과

- ✅ 한국 주식 6자리 코드 (예: 005930)
- ✅ Yahoo Finance 형식 (예: 005930.KS, 035720.KQ)
- ✅ 즐겨찾기에 등록된 58개 주요 종목
- ✅ 모든 에이전트의 프롬프트에서 종목명 표시
- ✅ 최종 의사결정 과정에서도 종목명 표시

## 장점

1. **가독성 향상**: 숫자만 보고는 어떤 종목인지 알기 어려웠던 문제 해결
2. **사용자 경험 개선**: 한국 사용자들이 익숙한 종목명으로 확인 가능
3. **일관성**: 모든 에이전트와 프롬프트에서 통일된 형식 사용
4. **확장성**: 즐겨찾기 파일에 종목 추가만 하면 즉시 반영

## 추가 개선 가능 사항

1. 더 많은 코스닥 종목 추가
2. ETF 종목명 지원
3. 종목명 캐싱 강화로 성능 개선
4. 영문명과 한글명 선택 옵션 제공
