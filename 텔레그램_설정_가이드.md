# 텔레그램 봇 설정 및 사용 가이드

## 1. 텔레그램 봇 생성 (5분 소요)

### Step 1: BotFather에서 봇 생성

1. 텔레그램에서 `@BotFather` 검색 후 대화 시작
2. `/newbot` 입력
3. 봇 이름 입력 (예: `내 주식 분석기`)
4. 봇 유저네임 입력 (예: `my_stock_analyzer_bot`) — 반드시 `_bot`으로 끝나야 함
5. 생성 완료 시 **API 토큰**이 표시됨 (예: `7123456789:AAFxxx...`)

```
이 토큰을 복사해둔다. 외부에 노출하면 안 된다.
```

### Step 2: Chat ID 확인

1. 생성한 봇에게 텔레그램에서 아무 메시지를 보낸다 (예: "hello")
2. 브라우저에서 아래 URL 접속 (토큰 부분을 자기 토큰으로 교체):

```
https://api.telegram.org/bot{여기에토큰}/getUpdates
```

3. JSON 응답에서 `"chat":{"id":123456789}` 부분의 숫자가 Chat ID

또는 터미널에서:

```bash
curl "https://api.telegram.org/bot{토큰}/getUpdates" | python3 -m json.tool | grep '"id"'
```

### Step 3: .env 파일에 입력

```bash
cd ~/stock_analyzer
nano .env
```

```env
TELEGRAM_BOT_TOKEN=7123456789:AAFxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
TELEGRAM_CHAT_ID=123456789
```

### Step 4: 연결 테스트

```bash
python main.py AAPL --telegram
```

텔레그램에 차트 이미지 + 분석 리포트가 도착하면 성공.

## 2. 사용 모드

### 2.1 단발성 전송 모드 (--telegram)

분석 실행 후 결과를 텔레그램으로 한 번 전송한다. cron 자동화에 적합.

```bash
# 기본 분석 → 텔레그램 전송
python main.py AAPL --telegram

# AI 분석 포함 → 텔레그램 전송
python main.py AAPL --ai gpt4o --telegram

# 계좌 크기 지정
python main.py NVDA --ai gpt4o --account 50000 --telegram
```

전송 내용:
1. 기술 분석 차트 이미지 (PNG)
2. 포맷팅된 텍스트 리포트 (가격, 지표, 리스크, 거시경제, 펀더멘털, AI 분석)
3. JSON 상세 데이터 파일

### 2.2 대화형 봇 모드 (--bot)

봇이 백그라운드에서 실행되며, 텔레그램에서 명령어를 보내면 분석을 수행한다.

```bash
# 봇 시작 (AI 없이)
python main.py --bot

# 봇 시작 (GPT-4o 분석 포함)
python main.py --bot --ai gpt4o --account 50000

# 백그라운드 실행
nohup python main.py --bot --ai gpt4o --account 50000 > bot.log 2>&1 &
```

텔레그램에서 사용 가능한 명령어:

| 명령어 | 기능 | 예시 |
|--------|------|------|
| `/analyze 티커` | 단일 종목 분석 | `/analyze AAPL` |
| `/a 티커` | 분석 (단축) | `/a NVDA` |
| `/watchlist 종목들` | 여러 종목 일괄 분석 | `/watchlist AAPL,MSFT,NVDA,TSLA` |
| `/help` | 사용법 안내 | `/help` |

## 4. cron + 텔레그램 자동화

매일 장 마감 후 자동 분석 → 텔레그램 전송:

```bash
crontab -e
```

```cron
# 매일 한국시간 06:05 (미국 장 마감 후)
5 6 * * 2-6 cd /home/사용자명/stock_analyzer && /home/사용자명/stock_analyzer/venv/bin/python main.py AAPL --ai gpt4o --account 50000 --telegram >> output/cron.log 2>&1

# 여러 종목 순차 전송
10 6 * * 2-6 cd /home/사용자명/stock_analyzer && for t in MSFT NVDA TSLA; do /home/사용자명/stock_analyzer/venv/bin/python main.py $t --ai gpt4o --account 50000 --telegram; sleep 5; done >> output/cron.log 2>&1
```

## 5. 봇 모드를 systemd 서비스로 등록

봇이 서버 재시작 후에도 자동 실행되게 한다:

```bash
sudo nano /etc/systemd/system/stock-bot.service
```

```ini
[Unit]
Description=Stock Analysis Telegram Bot
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=사용자명
WorkingDirectory=/home/사용자명/stock_analyzer
ExecStart=/home/사용자명/stock_analyzer/venv/bin/python main.py --bot --ai gpt4o --account 50000
Restart=always
RestartSec=10
StandardOutput=append:/home/사용자명/stock_analyzer/output/bot.log
StandardError=append:/home/사용자명/stock_analyzer/output/bot.log

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable stock-bot
sudo systemctl start stock-bot

# 상태 확인
sudo systemctl status stock-bot

# 로그 확인
journalctl -u stock-bot -f
```

## 6. 보안 주의사항

1. **봇 토큰**은 .env 파일에만 저장. git에 커밋하지 않는다. `.gitignore`에 `.env` 추가.
2. **Chat ID 검증**: 봇 모드에서 본인의 Chat ID만 허용하도록 구현되어 있다. 다른 사용자가 봇에 메시지를 보내도 무시된다.
3. **봇을 그룹에 추가하지 않는다.** 개인 채팅에서만 사용한다.
4. **.env 파일 권한 제한**: `chmod 600 .env`

## 7. 문제 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| 메시지가 안 옴 | 토큰 또는 Chat ID 오류 | .env 값 재확인. 봇에게 먼저 메시지 보낸 후 getUpdates로 ID 확인 |
| 차트 이미지가 안 옴 | 차트 생성 실패 | `plotly_get_chrome` 실행 또는 matplotlib fallback 확인 |
| 봇 모드에서 명령어 무반응 | Chat ID 불일치 | 봇에 보낸 메시지의 Chat ID와 .env의 TELEGRAM_CHAT_ID 일치 확인 |
| `Connection refused` | 텔레그램 API 차단 | 인터넷 연결 확인. 일부 네트워크에서 텔레그램 API 차단 가능 |
| 메시지가 잘림 | 4096자 초과 | 자동 분할 전송 구현됨. AI 분석이 매우 길 경우 여러 메시지로 분할 |
