# 미국 주식 AI 분석 시스템 - 원본 보고서 비판적 보완 가이드

## 1. 원본 보고서의 핵심 부족점 진단

### 1.1 치명적 결함: 리스크 관리 체계 완전 부재

원본 보고서는 데이터 수집과 분석에만 집중하고, **매매에서 가장 중요한 리스크 관리를 전혀 다루지 않는다.** 아무리 정교한 AI 분석이라도 리스크 관리 없이는 단 한 번의 대형 손실로 계좌가 파괴된다.

보완해야 할 리스크 관리 항목:

- **포지션 사이징**: 계좌 대비 단일 종목 최대 비중 (통상 2-5%)
- **손절매(Stop-Loss) 전략**: ATR 기반 동적 손절, 고정 비율 손절
- **최대 드로다운 허용치**: 전체 포트폴리오 기준 최대 허용 손실 (예: -15%)
- **Kelly Criterion 또는 변형 공식**: 베팅 크기의 수학적 산출
- **상관관계 리스크**: 동일 섹터/테마 과집중 방지

### 1.2 백테스팅/검증 프레임워크 부재

보고서는 AI가 분석하고 조언을 내리는 것까지만 다루고, **해당 전략이 과거에 유효했는지 검증하는 과정이 없다.** 백테스팅 없는 전략은 도박과 다르지 않다.

필수 검증 체계:

- **과거 데이터 백테스팅**: 최소 5년 이상의 히스토리컬 데이터로 전략 검증
- **워크포워드 분석**: In-Sample/Out-of-Sample 분리하여 과적합(Overfitting) 방지
- **몬테카를로 시뮬레이션**: 수천 번의 랜덤 시나리오로 전략의 강건성 검증
- **성과 지표**: Sharpe Ratio, Sortino Ratio, Max Drawdown, Win Rate, Profit Factor

### 1.3 실행 비용 및 슬리피지 무시

실제 매매에서 발생하는 비용을 완전히 무시했다:

- **슬리피지**: 주문 체결 시 예상 가격과 실제 체결 가격 차이
- **스프레드**: Bid-Ask 스프레드로 인한 즉시 손실
- **거래 수수료**: 빈번한 매매 시 누적 비용
- **세금**: 미국 주식 단기 매매 시 양도소득세 (단기: 최대 37%, 장기: 0-20%)

### 1.4 센티먼트/대안 데이터 부족

보고서는 "뉴스 심리"를 언급하지만 구체적 구현이 없다:

- **옵션 시장 데이터**: IV(내재변동성), Put/Call Ratio, 옵션 체인 - 기관 투자자의 포지션 방향 파악
- **공매도 데이터**: Short Interest, Days to Cover
- **내부자 거래**: SEC Form 4 기반 경영진 매매 동향
- **소셜 센티먼트**: Reddit(r/wallstreetbets), Twitter/X 언급량 및 감성 분석
- **다크풀 데이터**: 기관의 대량 거래 흐름

### 1.5 포트폴리오 레벨 분석 부재

개별 종목 분석만 다루고 포트폴리오 차원의 분석이 없다:

- **섹터 로테이션**: 경기 사이클별 유망 섹터 전환
- **상관관계 매트릭스**: 보유 종목 간 상관관계로 실질 분산 투자 여부 확인
- **베타 노출**: 포트폴리오 전체의 시장 민감도
- **팩터 노출**: 모멘텀, 밸류, 퀄리티 등 스타일 팩터 편향

### 1.6 VISTA 프레임워크의 실체 불명확

보고서에서 언급한 "VISTA(Vision-Language Inference)" 프레임워크는 2026년 4월 기준 널리 인정된 표준 프레임워크가 아니다. 멀티모달 LLM에 차트를 입력하는 것은 유효하나, 이를 특정 프레임워크 명칭으로 포장한 것은 과장된 표현일 수 있다. 실질적으로는 GPT-4o, Claude 3.5 Sonnet 등의 Vision 기능을 직접 활용하는 것이 현실적이다.

### 1.7 에러 처리 및 시스템 장애 대응 없음

- API Rate Limit 초과 시 대응
- 데이터 결측/이상치 처리
- 네트워크 장애 시 fallback
- 장 중 시스템 다운 시 열린 포지션 관리

## 3. AI 모델 비교: OpenAI GPT-4o vs 로컬 LLM

### 3.1 OpenAI GPT-4o

| 항목 | 내용 |
|------|------|
| **장점** | 멀티모달(이미지+텍스트) 네이티브 지원. 차트 이미지 분석 가능. 추론 능력 최상급. API 안정성 높음 |
| **단점** | 비용 발생 ($2.50/1M input, $10/1M output). 데이터가 OpenAI 서버 경유. 레이턴시 존재. Rate Limit |
| **월 예상 비용** | 일 10회 분석 기준 약 $5-15/월 (입력 토큰량에 따라 변동) |
| **차트 분석** | Vision 기능으로 캔들차트, 패턴 직접 인식 가능 |
| **적합한 경우** | 멀티모달 분석이 핵심이고, 구축 속도가 중요할 때 |

### 3.2 로컬 LLM (Ollama 기반)

| 항목 | 내용 |
|------|------|
| **장점** | 완전 무료 (전기세 제외). 데이터 외부 유출 없음. Rate Limit 없음. 커스터마이징 자유 |
| **단점** | GPU 필요 (최소 VRAM 8GB, 권장 16GB+). 멀티모달 지원 모델 제한적. 추론 품질이 GPT-4o 대비 낮음 |
| **추천 모델** | llama3.1:70b (텍스트), llava:34b (멀티모달), deepseek-coder-v2 (코드/수치) |
| **하드웨어 요구** | 70B 모델: RTX 4090(24GB) 이상 권장. 8B 모델: RTX 3060(12GB) 가능 |
| **적합한 경우** | 프라이버시 중시, 대량 반복 분석, GPU 보유 시 |

### 3.3 하이브리드 전략 (추천)

실전에서 가장 현실적인 접근:

1. **데이터 정제/구조화**: 로컬 LLM (비용 절감, 단순 작업)
2. **차트 이미지 분석**: GPT-4o (멀티모달 품질이 압도적)
3. **최종 종합 판단**: GPT-4o 또는 Claude (복잡한 추론)
4. **뉴스 요약**: 로컬 LLM (대량 처리, 비용 절감)

## 5. 구현 로드맵 (단계별)

### Phase 1: 데이터 파이프라인 (1-2주)
- yfinance + FRED API 연동
- OHLCV 데이터 수집 자동화
- 기술 지표 계산 모듈 (pandas-ta)
- 데이터 품질 검증 로직

### Phase 2: AI 분석 엔진 (2-3주)
- 차트 이미지 자동 생성 (Plotly)
- LLM 프롬프트 템플릿 설계
- GPT-4o Vision 차트 분석 연동
- 로컬 LLM(Ollama) 텍스트 분석 연동

### Phase 3: 리스크 관리 (1-2주)
- 포지션 사이징 모듈
- 손절/익절 자동 계산
- 포트폴리오 상관관계 분석
- 드로다운 모니터링

### Phase 4: 백테스팅 (2-3주)
- 히스토리컬 데이터 기반 전략 검증
- 성과 지표 대시보드
- 워크포워드 분석

### Phase 5: 자동화 및 알림 (1주)
- cron 기반 스케줄링 (Ubuntu)
- 텔레그램 봇 알림

## 6. 원본 보고서의 기타 문제점

1. **n8n 의존성 과대평가**: n8n은 로우코드 도구로 프로토타이핑에 적합하나, 복잡한 데이터 파이프라인에서는 Python 직접 구현이 디버깅/커스터마이징 측면에서 우월하다. Ubuntu 환경이라면 Python + cron이 더 적합하다.

2. **CrewAI 언급의 실효성**: 멀티 에이전트 프레임워크는 개념적으로 매력적이나, 개인 투자자 수준에서는 오버엔지니어링이다. 단일 파이프라인 스크립트가 유지보수와 디버깅에서 훨씬 효율적이다.

3. **MCP 서버 구축의 현실성**: MCP는 AI 도구 연동에 유용하나, 주식 분석 시스템의 핵심이 아니다. 데이터 파이프라인을 먼저 구축하고, MCP는 선택적 확장으로 접근해야 한다.

4. **TradingView MCP Server**: TradingView의 ToS(이용약관)상 자동화된 스크래핑/캡처는 위반 소지가 있다. Plotly로 자체 차트를 생성하는 것이 법적/기술적으로 안전하다.

5. **실시간 데이터의 함정**: 개인 투자자에게 실시간 데이터는 과매매(Overtrading)를 유발할 수 있다. 일봉 기반 EOD(End of Day) 분석이 개인 투자자에게 더 적합한 경우가 많다.
