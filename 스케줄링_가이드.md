# 다수 종목 스케줄링 가이드

## 방법 1: 단일 종목 직접 실행

```bash
cd ~/stock_auto/stock_analyzer
python main.py AAPL --ai ollama --telegram
```

---

## 방법 2: --watchlist 옵션 (파일 참조)

`watchlist.txt` 파일에서 종목을 읽어 일괄 분석한다.

### watchlist.txt 형식

```
# 주석은 #으로 시작
# 빈 줄은 무시됨

AAPL
MSFT
NVDA
# TSLA   ← 주석 처리하면 제외됨
```

### 실행

```bash
# 기본 파일 (watchlist.txt) 사용
python main.py --watchlist

# 다른 파일 지정
python main.py --watchlist my_tickers.txt

# AI 분석 + 텔레그램 조합
python main.py --watchlist --ai ollama --telegram
python main.py --watchlist growth_stocks.txt --ai gpt4o --telegram
```

### 용도별 파일 분리 예시

```
watchlist.txt          # 기본 관심 종목
growth_stocks.txt      # 성장주
dividend_stocks.txt    # 배당주
etf_list.txt           # ETF
```

---

## 방법 3: 셸 스크립트 (run_watchlist.sh)

`run_watchlist.sh`는 가상환경 활성화 + 파일 참조 + 로그를 한번에 처리한다.

### 사용법

```bash
# 기본 실행 (watchlist.txt, AI 없음, 텔레그램 없음)
./run_watchlist.sh

# 인자: [종목파일] [AI모드] [텔레그램여부]
./run_watchlist.sh watchlist.txt ollama true
./run_watchlist.sh growth_stocks.txt gpt4o true
./run_watchlist.sh etf_list.txt none false
```

---

## 방법 4: crontab 스케줄링 (자동 실행)

### 등록

```bash
crontab -e
```

### crontab 예시

```cron
# ┌───── 분 (0-59)
# │ ┌───── 시 (0-23)
# │ │ ┌───── 일 (1-31)
# │ │ │ ┌───── 월 (1-12)
# │ │ │ │ ┌───── 요일 (0-7, 0과 7은 일요일)
# │ │ │ │ │

# 평일 오전 7시 (KST) = 미국 장 마감 후
0 7 * * 1-5 /home/ubuntu/stock_auto/stock_analyzer/run_watchlist.sh watchlist.txt ollama true >> /home/ubuntu/stock_auto/stock_analyzer/output/cron.log 2>&1

# 주말 오전 10시 주간 리뷰 (다른 종목 파일)
0 10 * * 6 /home/ubuntu/stock_auto/stock_analyzer/run_watchlist.sh etf_list.txt gpt4o true >> /home/ubuntu/stock_auto/stock_analyzer/output/cron_weekly.log 2>&1
```

### 확인/관리

```bash
# 등록된 스케줄 확인
crontab -l

# 로그 확인
tail -f ~/stock_auto/stock_analyzer/output/cron.log

# 스케줄 삭제
crontab -e   # 해당 줄 삭제 후 저장
```

---

## 참고

- 종목 간 5초 딜레이가 자동 적용됨 (API 부하 방지)
- `watchlist.txt`는 `stock_analyzer/` 디렉토리에 위치
- `#`으로 주석 처리하면 해당 종목은 분석에서 제외됨
- crontab 경로는 반드시 절대 경로 사용
