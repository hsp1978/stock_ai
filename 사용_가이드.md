# 주식 AI 분석 시스템 - 실전 사용 가이드

## 1. 일상적인 매매 워크플로우

### 1.1 장 마감 후 루틴 (가장 기본적인 사용법)

미국 장 마감 (한국시간 06:00) 이후 관심 종목을 분석한다.

```bash
cd ~/stock_analyzer
source venv/bin/activate

# 기본 분석 (AI 없이, 가장 빠름)
python main.py AAPL

# AI 분석 포함 (GPT-4o가 차트까지 읽고 판단)
python main.py AAPL --ai gpt4o --account 50000
```

실행하면 아래 순서로 진행된다:

```
[1/6] 데이터 수집       → yfinance에서 시세/재무/옵션 데이터 수집
[2/6] 기술 지표 계산     → RSI, MACD, 볼린저밴드, ADX 등 자동 계산
[3/6] 리스크 분석        → 손절가, 익절가, 매수 수량 자동 산출
[4/6] 차트 생성          → 기술 분석 차트 PNG 이미지 생성
[5/6] AI 분석            → 수치+차트 이미지를 AI에 전송하여 종합 판단
[6/6] 리포트 저장        → output/ 폴더에 JSON + TXT 리포트 저장
```

### 1.2 출력 파일 위치

```
output/
├── AAPL_analysis.png              # 기술 분석 차트 이미지
├── AAPL_report_20260403.json      # 구조화된 전체 데이터 (프로그래밍용)
└── AAPL_report_20260403.txt       # 사람이 읽는 텍스트 리포트
```

- **PNG**: 직접 눈으로 차트를 확인할 때 사용. AI에게도 이 이미지가 전달됨
- **JSON**: 다른 프로그램에서 데이터를 가져다 쓸 때 사용
- **TXT**: 터미널에서 바로 읽거나, 카톡/메모에 복붙할 때 사용

## 2. 시나리오별 사용법

### 2.1 "이 종목 지금 사도 되나?" — 단일 종목 판단

```bash
python main.py NVDA --ai gpt4o --account 30000
```

확인할 항목:
1. **콘솔의 RSI 값** → 70 이상이면 과매수, 30 이하면 과매도
2. **추세 강도(ADX)** → 25 이상이면 추세 존재, 20 이하면 횡보
3. **손절가/익절가** → 감당 가능한 손실인지 확인
4. **AI 분석 결과** → "매수/매도/관망" 판단과 신뢰도 확인

### 2.2 "관심 종목 5개 한번에 봐줘" — 일괄 분석

`batch_analyze.sh` 파일을 만든다:

```bash
#!/bin/bash
# batch_analyze.sh
TICKERS="AAPL MSFT NVDA GOOGL TSLA"
ACCOUNT=50000

for t in $TICKERS; do
    echo ""
    echo "============================================"
    echo "  $t 분석"
    echo "============================================"
    python main.py $t --ai gpt4o --account $ACCOUNT
    sleep 3
done
```

```bash
chmod +x batch_analyze.sh
./batch_analyze.sh
```

분석이 끝나면 `output/` 폴더에 종목별 리포트가 저장된다.

### 2.3 "AI 없이 빠르게 숫자만 보고 싶다" — 기본 모드

```bash
python main.py META
```

`--ai` 옵션 없이 실행하면 AI 호출을 건너뛴다. 데이터 수집 → 지표 계산 → 리스크 분석 → 차트까지만 수행하며, 보통 5-10초 내에 완료된다. API 비용도 0원.

### 2.4 "GPU 있으니까 로컬에서 돌리고 싶다" — Ollama 모드

```bash
# Ollama 서버가 실행 중인지 확인
curl http://localhost:11434/api/tags

# 실행
python main.py AAPL --ai ollama --account 30000
```

Ollama 모드는 차트 이미지 분석이 안 된다. 텍스트 수치 데이터만으로 분석한다. 비용은 0원이지만 GPT-4o 대비 분석 품질이 떨어진다.

## 3. 콘솔 출력 읽는 법

실제 실행 시 아래와 같은 출력이 나온다:

```
============================================================
  AAPL 종합 분석 시작 - 2026-04-03 18:30
============================================================

[1/6] 데이터 수집 중...
  ✓ OHLCV: 503일 데이터 수집 완료          ← 약 2년치 일봉 데이터
  ✓ 펀더멘털: Apple Inc.                   ← 기업명 + 재무제표 수집 완료
  ✓ 옵션: P/C Ratio(Vol)=0.82             ← 콜 우위 (시장 낙관적)
  ✓ 거시경제: 7개 지표 수집                 ← 금리, CPI, VIX 등
  ✓ 내부자 거래: 최근 5건                   ← 경영진 매매 동향
```

```
[2/6] 기술 지표 계산 중...
  ✓ 현재가: $178.52
  ✓ RSI: 62.3                             ← 중립 (30-70 사이)
  ✓ 추세 강도: strong                      ← ADX > 25, 추세 유효
```

```
[3/6] 리스크 분석 중...
  ✓ 손절가: $170.14 (4.69%)               ← 이 가격 이하로 떨어지면 손절
  ✓ 익절가: $195.28 (9.39%)               ← 목표 수익 달성 가격
  ✓ 권장 수량: 23주 ($4,105.96)            ← 계좌 대비 적정 매수량
  ⚠ 손절 거리(4.69%)가 과도하게 넓음        ← 경고 메시지 (있을 때만 출력)
```

### 경고 메시지 의미

| 메시지 | 의미 | 대응 |
|--------|------|------|
| `⚠ 포지션 비중 초과` | 1종목에 계좌의 5% 이상 들어감 | 매수 수량 줄이기 |
| `⚠ 손절 거리가 과도하게 넓음` | 변동성이 커서 손절폭이 10% 초과 | 소량 매수하거나 관망 |
| `⚠ 매매 불가` | 계좌가 너무 작아서 1주도 못 삼 | `--account` 값 확인 |

## 4. AI 분석 결과 읽는 법

GPT-4o 분석을 사용하면 아래와 같은 형태의 결과가 출력된다:

```
## 종합 판단
매수 (신뢰도: 7/10)                        ← 핵심 결론

## 기술적 분석
- 50일 SMA 위에서 가격이 유지되고 있어 상승 추세 유효
- RSI 62로 과매수 구간 진입 전이며 추가 상승 여력 존재
- MACD 히스토그램이 양전환되어 모멘텀 회복 신호
- 볼린저 밴드 중심선 위에 위치하여 강세 편향

## 펀더멘털 분석
- Forward P/E 28.5배로 5년 평균(27배) 대비 소폭 고평가
- 영업이익률 30.2%로 업종 최상위
- 잉여현금흐름 $110B으로 자사주 매입 여력 충분

## 리스크 관리
- 진입가: $178.52
- 손절가: $170.14 (ATR 2배 기준)
- 익절가: $195.28 (R:R = 1:2)
- 권장 비중: 계좌의 4.99% (23주)

## 시장 환경
- 10Y 국채 4.2%로 기술주에 부정적 압력 존재
- VIX 18.5로 변동성 보통 수준
- 연준 금리 인하 기대감은 소폭 긍정적

## 핵심 리스크 요인
- 금리 인상 재개 시 성장주 전반 하락 가능
- 중국 규제 리스크 (매출 비중 19%)
- AI 투자 대비 수익화 속도 부진 시 실망 매물 가능
```

**읽는 순서:**
1. "종합 판단"에서 매수/매도/관망과 신뢰도를 먼저 확인
2. 신뢰도 7 이상이면 유의미한 신호, 5 이하면 불확실
3. "핵심 리스크 요인"을 반드시 읽고, 감당 가능한지 판단
4. "리스크 관리" 수치를 실제 주문에 그대로 활용

## 5. 리포트 수치를 실제 매매에 적용하는 법

### 5.1 매수 진입 시

리포트에서 나온 수치를 실제 증권사 주문에 이렇게 적용한다:

```
리포트 출력:
  손절가: $170.14
  익절가: $195.28
  권장 수량: 23주

실제 주문:
  1) 시장가 또는 지정가($178.52)로 23주 매수
  2) 즉시 Stop-Loss 주문: $170.14에 23주 매도 (손절)
  3) Limit 매도 주문: $195.28에 23주 매도 (익절)
```

### 5.2 분할 매수/매도 전략

한 번에 전량 진입하지 않고 나눠서 진입하는 것을 권장한다:

```
권장 수량 23주 기준:
  1차 진입: 12주 (현재가 $178.52)
  2차 진입: 11주 (지지선 근처 $174 도달 시)

또는:
  1차 진입: 8주 (50%)
  2차 진입: 8주 (1차 매수 후 +2% 돌파 확인 시)
  3차 진입: 7주 (추세 지속 확인 시)
```

### 5.3 손절 후 대응

```
손절 발동 ($170.14 도달):
  → 즉시 포지션 정리
  → 최소 3-5 거래일 대기 (감정적 재진입 방지)
  → 다시 main.py 실행하여 새 분석 확인
  → 새 분석에서 매수 신호가 나올 때만 재진입
```

```python
SMA_PERIODS = [5, 10, 20]        # 짧은 이평선

DEFAULT_TAKE_PROFIT_RATIO = 1.5   # R:R 1:1.5
DEFAULT_HISTORY_PERIOD = "6mo"    # 6개월 데이터면 충분
```

**스윙 (보유 1-4주) — 기본값**
```python
SMA_PERIODS = [20, 50, 200]

DEFAULT_STOP_LOSS_ATR_MULT = 2.0
```

**장기 투자 (보유 1개월+)**

## 7. 차트 이미지 해석법

생성되는 차트는 4개 패널로 구성된다:

```
┌──────────────────────────────────────────┐
│  [1] 가격 + 이동평균선 + 볼린저 밴드      │  ← 추세와 가격 위치 파악
│      노란선: SMA 20 (단기)               │
│      주황선: SMA 50 (중기)               │
│      분홍선: SMA 200 (장기)              │
│      하늘색 영역: 볼린저 밴드             │
├──────────────────────────────────────────┤
│  [2] 거래량 바 + 20일 평균선              │  ← 거래량 급증 = 의미 있는 움직임
├──────────────────────────────────────────┤
│  [3] RSI (14일)                          │  ← 70 위: 과매수, 30 아래: 과매도
│      빨간 점선: 70 (과매수 기준)          │
│      녹색 점선: 30 (과매도 기준)          │
├──────────────────────────────────────────┤
│  [4] MACD + 시그널 + 히스토그램           │  ← 교차 = 추세 전환 신호
│      파란선: MACD                        │
│      빨간선: 시그널                       │
│      막대: 히스토그램 (양수=상승모멘텀)     │
└──────────────────────────────────────────┘
```

**핵심 패턴:**

| 차트에서 보이는 것 | 의미 | 행동 |
|---------------------|------|------|
| 가격이 SMA 200 위, SMA 20 > SMA 50 | 건강한 상승 추세 | 매수 검토 |
| 가격이 볼린저 하단 터치 + RSI < 30 | 과매도 반등 가능 | 매수 검토 |
| 가격이 볼린저 상단 돌파 + RSI > 70 | 과매수 또는 강한 모멘텀 | 신규 매수 주의 |
| MACD가 시그널 위로 교차 (골든크로스) | 상승 모멘텀 시작 | 매수 신호 |
| MACD가 시그널 아래로 교차 (데드크로스) | 하락 모멘텀 시작 | 매도/관망 신호 |
| 거래량 급증 + 가격 상승 | 세력/기관 매집 가능 | 추세 추종 매수 |
| 거래량 급증 + 가격 하락 | 대량 투매 | 관망 또는 손절 확인 |

## 9. 주의사항

1. **이 시스템은 투자 조언 도구가 아니다.** AI의 분석은 참고 자료일 뿐, 최종 매매 판단은 본인이 한다.
2. **장 중 실시간 데이터가 아니다.** yfinance는 약 15-20분 지연 데이터를 제공한다. 장 마감 후 EOD 분석에 최적화되어 있다.
3. **AI 분석 신뢰도 5 이하는 무시한다.** 데이터가 불충분하거나 혼조세일 때 AI도 확신이 없다. 이때는 관망이 정답.
4. **손절가는 반드시 지킨다.** 시스템이 계산한 손절가를 무시하고 버티는 것은 리스크 관리 체계를 무력화하는 행위다.
5. **하나의 분석만으로 올인하지 않는다.** 여러 종목을 분석하고, 가장 확신이 높은 1-2개에 분산 진입한다.
